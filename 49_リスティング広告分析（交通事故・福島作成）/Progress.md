## 1. 目的（このプロジェクトのゴール）

`shunta.fukushima@legalaccess.co.jp` の同一Googleログインで運用している広告について、
**会社（リーガルアクセス株式会社）の広告費は会社カードで、法律事務所の広告費は事務所カードで**
支払えるようにする。

### 結論として採った方式
- Google広告は「キャンペーンごとに支払カードを分ける」ことは**できない**（支払いはアカウント単位）。
- そこで **広告アカウントを2つに分ける**方式を採用（道A）。
  - 既存アカウント `518-401-4195` … **事務所**用として残す（事務所キャンペーン群）
  - 新規アカウント `795-110-4182` … **会社（リーガルアクセス）**用として新設し、会社カードを紐付け
- 会社の広告は「**法務生成AI**」キャンペーン1つのみ。これを518から795へ移す（＝795で作り直す）。

## 5. 再開方法（次チャットのための手順メモ）

### ブラウザ操作（Web版Google広告）の再開 ← 作業B・C・Dはこれで進める
- ログイン状態は `C:\Users\shunt\AppData\Local\Temp\gads-state.json` に保存済み（storageState方式）。専用プロファイル `C:\Users\shunt\AppData\Local\Temp\gads-drive` を使う独立Chrome。
- **操作方式（CLAUDE.mdに記録済みの教訓）**：
  - ユーザーの現用Chrome/プロファイルを直接奪わない。**独立した別ブラウザ＋手動ログイン1回＋storageState保存**で操作する。
  - 使い捨てスクリプトは `#old/` 配下（`_gads_drive.js` がドライバ。`_cmd.json` にコマンドを書き `_out.log` で結果確認するRPC方式）。
  - 画面確認は `#old/_*.png` のスクショ。
- **ドライバ起動の実手順（2026-06-25 実証）**：
  1. `cd "#old"` して `node _gads_drive.js > _driver_stdout.log 2>&1 &`（バックグラウンド起動）。約10秒待つ。
  2. `_out.log` の末尾を見る。`LOGGED_IN false` でも慌てない——**URLが `nav/selectaccount`（アカウント選択画面）なら、それはログイン済みでアカウント未選択なだけ**。ドライバの `LOGGED_IN` 判定は `/aw/` がURLに無いと false を返す誤判定をする。`{"action":"read"}` を `_cmd.json` に書いて本文を読み、`shunta.fukushima@legalaccess.co.jp` とアカウント一覧（518 / 795）が出ていればログインは生きている。
  3. 795を開くには `{"action":"click_text","text":"795-110-4182","wait":7000}` → 遷移先URL末尾が `__e=7951104182` になれば795。`{"action":"read"}` で中身確認。
  4. **コマンド例**：`goto`（URL遷移）／`shot`（スクショ）／`click_text`（テキスト一致クリック）／`click_role`／`fill`（ラベル一致入力）／`read`（URL・タイトル・本文2000字出力）／`quit`。1コマンドごとに `_cmd.json` を書き、`_out.log` 末尾で結果確認する半自動方式。
- **作業Aの送信成功は、このドライバでWeb版795を開いて実証済み**（795キャンペーン画面に法務生成AIが1件実在）。同じ手順で作業B（アカウント名変更）・C（518の支払いカード確認）に入れる。

### Ads Editorの操作（作業Aで使用。今後は基本Web版でよい）
- 起動：`C:\Users\shunt\AppData\Local\Google\Google Ads Editor\google_ads_editor_launcher.exe`。または `#old/_launch_editor.ps1`。
- **Ads Editorは UI Automation（UIA）にコントロールを公開している**（Qt製だが操作可）。`#old/_*.ps1` 群で、左ツリー・ボタン・データグリッドを名前で取得できる。ただし**PowerShellスクリプトは必ずUTF-8 BOM付きで書く**こと（日本語の正規表現・文字列がShift-JS誤読でパースエラーになる。Pythonで `codecs.BOM_UTF8` を先頭に付けて書き出すのが確実）。
  - UIA読み取りの主な手がかり：左カテゴリツリーの「キャンペーン (N)」のNで、いま右ペインが518（6件）か795（1件）かを判定できる。ボタンは `送信`／`変更内容を確認`／`最新の変更を取得` が名前付きで取れる。
- **⚠️ Ads Editorの「クリック操作」はこの環境では危険**：UIAの `.Select()` はツリーのハイライトを変えるだけで右ペインが切り替わらず、実際の選択切替には**マウス座標クリック**が要る。だがバックグラウンドのスクリプトからは**Windowsの前面化ロックを越えてAds Editorを安定前面化できず**、マウスが裏のVS Code／Chromeに誤爆する（2026-06-25に実際に複数回誤爆）。`WScript.Shell.AppActivate` も「成功(True)」を返しつつ実際は別窓が前面、という嘘をつく。
  - **対策＝送信などの最終GUIクリックは福島さん本人に渡す**。Claudeは①Ads Editor起動②UIAで中身を読んで送信前チェック③送信範囲の確定、までやり、送信ボタンの最終クリックだけ本人に依頼する（CLAUDE.md「最後の一歩」ルール）。誤爆ガード（クリック直前に `GetForegroundWindow()==target` を確認し、違えばクリック中止）を入れたスクリプトは作ったが、結局この環境では前面化できずクリックは常に中止された＝Claude単独でのAds Editor送信は不可と確定。
